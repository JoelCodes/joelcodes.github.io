# Requirements: Joel Shinness Website

**Defined:** 2026-02-09
**Core Value:** Small business owners can understand what Joel does, trust his process, and easily reach out to start a conversation.

## v1.2 Requirements

Requirements for Homepage Refinement milestone. Each maps to roadmap phases.

### Foundation

- [x] **FOUND-01**: Icon library migrated from lucide-static to @lucide/astro for tree-shaking ✓
- [x] **FOUND-02**: Isometric CSS utilities added to global.css (transform-style: preserve-3d patterns) ✓
- [x] **FOUND-03**: Shadow utilities for isometric elements support dark mode glow transformation ✓

### Hero

- [x] **HERO-01**: Hero headline reframes from "what I build" to "outcomes you get" (time saved, money saved, customers satisfied) ✓
- [x] **HERO-02**: Visual outcome badges display up to 3 key metrics below headline ✓
- [x] **HERO-03**: Badges use design system shadow utilities and pass WCAG contrast checks ✓

### Process

- [x] **PROC-01**: Each of 5 process steps has 1-2 sentence description explaining what happens ✓
- [x] **PROC-02**: Each process step has an isometric mini-illustration ✓
- [x] **PROC-03**: Illustrations follow consistent style guide (45° light angle, neobrutalist palette, flat colors) ✓
- [x] **PROC-04**: Illustrations are optimized (<20KB each) and have descriptive alt text ✓

### Technology

- [x] **TECH-01**: Technology section reorganized into 3 categories: AI, Automations, Web Apps ✓
- [x] **TECH-02**: Each category has 1-2 sentence description explaining the offering ✓
- [x] **TECH-03**: Each category has an isometric illustration matching process section style ✓
- [x] **TECH-04**: Grid layout responsive (stacked mobile, 3-column desktop) ✓

### FAQ

- [x] **FAQ-01**: Dedicated /faq page created with existing FAQ content ✓
- [x] **FAQ-02**: FAQ page includes JSON-LD structured data for SEO ✓
- [x] **FAQ-03**: Navigation updated to include FAQ link ✓
- [x] **FAQ-04**: Footer FAQ section removed or replaced with link to /faq page ✓
- [x] **FAQ-05**: FAQ accordion passes keyboard accessibility (Enter/Space toggle, Tab navigation) ✓

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Multiple CTAs in hero | Reduces conversions by 266% per research |
| Animated isometric scenes | File size bloat, accessibility issues, static aesthetic is on-brand |
| Pricing calculator | Custom work varies too much, creates false expectations |
| Real-time chat widget | Overkill for freelance, contact form + FAQ covers communication |
