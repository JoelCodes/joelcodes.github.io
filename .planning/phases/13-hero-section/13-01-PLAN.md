  <name>Task 1: Create Badge.astro component</name>
  <files>src/components/ui/Badge.astro</files>
  <action>
Create a reusable outcome badge component with TypeScript Props interface:

**Props interface:**
```typescript
interface Props {
  label: string;           // Accessible label for screen readers (aria-label)
  value: string;           // The metric value displayed (e.g., "500+ hours")
  description?: string;    // Optional subtext (e.g., "saved per year")
  variant?: 'yellow' | 'turquoise' | 'magenta';
}
```

**Implementation requirements:**
1. Use `role="group"` with `aria-label={label}` for accessibility (WCAG 1.3.1)
2. Apply 3px neobrutalist border using `border-[3px] border-text-light dark:border-text-dark`
3. Use `iso-shadow` utility from global.css (transforms to glow in dark mode)
4. Map variants to WCAG-compliant text colors:
   - yellow: `text-yellow-text dark:text-yellow-text-dark`
   - turquoise: `text-turquoise-text dark:text-turquoise-text-dark`
   - magenta: `text-text-light dark:text-text-dark` (neutral, no accent)
5. Value text: `text-2xl md:text-3xl font-heading font-bold uppercase`
6. Description text: `text-sm md:text-base font-body text-text-muted-light dark:text-text-muted-dark`
7. Padding: `px-6 py-4` for comfortable spacing
8. Layout: `inline-flex flex-col items-center justify-center`
9. Default variant: 'yellow'

**Structure:**
```html
<div class="..." role="group" aria-label={label}>
  <span class="...value classes...">{value}</span>
  {description && <span class="...desc classes...">{description}</span>}
</div>
```

Follow Button.astro patterns for Props interface and default value handling.
