<task type="checkpoint:human-verify" gate="blocking">
  <what-built>
Complete Phase 6 performance and SEO optimization:
- SEO meta tags on all pages (title, description, og:image, Twitter Cards)
- JSON-LD Person structured data
- Automatic sitemap.xml generation
- Automatic robots.txt generation
- Image lazy loading optimization
- Font preloading
- Lighthouse CI enforcement in GitHub Actions
- PR preview deployments
  </what-built>
  <how-to-verify>
**1. SEO Meta Tags (view-source or DevTools)**
- Visit http://localhost:4321 (run `npm run dev` first)
- View page source (Ctrl+U / Cmd+Option+U)
- Verify these meta tags exist:
  - `<title>` contains "| Joel Shinness"
  - `<meta name="description"` exists
  - `<meta property="og:title"` exists
  - `<meta property="og:image"` exists
  - `<meta name="twitter:card"` exists
  - `<script type="application/ld+json"` contains Person schema
- Check other pages (Portfolio, Contact, Blog) have similar tags

**2. Sitemap and Robots.txt**
- Run `npm run build`
- Check `dist/sitemap-index.xml` exists
- Check `dist/robots.txt` exists and contains Sitemap: URL
- Preview: `npm run preview` then visit:
  - http://localhost:4321/sitemap-index.xml
  - http://localhost:4321/robots.txt

**3. Image Loading**
- Visit http://localhost:4321/portfolio
- Open DevTools Network tab, filter by images
- Scroll down slowly
- Verify images load as you scroll (lazy loading working)

**4. Lighthouse Score**
- Open DevTools > Lighthouse tab
- Run audit on Mobile preset
- Verify Performance score is 90+
- Check for any critical warnings

**5. GitHub Actions (visual check)**
- Review .github/workflows/deploy.yml contains lighthouse-ci-action
- Review .github/workflows/pr-preview.yml exists
- Review lighthouserc.json has minScore: 0.9

**Expected Results:**
- All meta tags present on all pages
- Sitemap lists all public pages
- Robots.txt allows crawling with sitemap reference
- Images lazy load below fold
- Lighthouse Performance 90+ on mobile
